Programação Orientada a Objeto

Introdução a Poo
A Programação Orientada a Objetos é um paradigma de programação que se baseia no conceito de "objetos". Cada objeto é uma instância de uma classe e possui atributos (dados) e métodos (ações). Esses objetos interagem entre si por meio de mensagens.

Princípios da POO:

  • Encapsulamento: Oculta a implementação interna dos objetos, permitindo o acesso apenas às interfaces públicas.

  • Herança: Permite criar novas classes a partir de classes existentes, herdando seus atributos e métodos.

  • Polimorfismo: Permite que objetos de diferentes classes respondam a uma mesma mensagem de forma específica.

    • Classes e Objetos:

    Classes: São as "receitas" para criar objetos. Definem a estrutura e o comportamento dos objetos.

    Objetos: São instâncias de classes. Representam entidades reais ou abstratas do mundo.

    Atributos e Métodos:


  • Atributos: São as características ou dados dos objetos, armazenando informações sobre o estado do objeto.

  • Métodos: São as ações que os objetos podem realizar, definindo seu comportamento.

    • Polimorfismo e Herança:


  • Polimorfismo: Permite tratar objetos de diferentes classes de forma uniforme, desde que implementem a mesma interface.

  • Herança: Permite que uma classe herde atributos e métodos de outra, facilitando a reutilização de código e estabelecendo hierarquias.
    • Essa é uma breve explicação sobre a Programação Orientada a Objetos. Esse paradigma é amplamente utilizado na indústria de software por sua capacidade de modelar o mundo real e facilitar o desenvolvimento e manutenção de sistemas complexos.
